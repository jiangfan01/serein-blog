# Frontend Career Agent 项目规划书

> 面向前端开发者的 AI 求职与成长助手

## 项目定位

**一句话描述**：帮助前端开发者学习技术、优化简历、匹配岗位、规划职业路径的垂直 AI Agent。

**核心价值**：
- 不是"什么都能聊"的通用 AI，而是专注前端领域的深度助手
- 回答有依据、可追溯，不乱编
- 从学习到求职全链路覆盖

**目标用户**：
- 前端初中级开发者（1-5 年经验）
- 正在求职或准备跳槽的前端
- 想系统学习前端知识的开发者

**向量化优化**：

#### 1.2 RAG 准确性提升

**检索优化**：
- [ ] Query 改写（用户问题 → 检索友好的 query）
- [ ] 多路召回（向量检索 + BM25 关键词检索）
- [ ] 重排序（Reranker 对检索结果二次排序）
- [ ] 检索结果过滤（相似度阈值、去重）

**回答生成优化**：
- [ ] 置信度判断机制
  - `high`：检索到 3+ 高相关文档
  - `medium`：检索到 1-2 个相关文档
  - `low`：检索结果相关性不足
  - `none`：没有检索到相关内容
- [ ] 引用来源展示（文章标题、链接、相关片段）
- [ ] 模型补充标记（区分知识库内容 vs 模型通用知识）

**输出结构化**：
```typescript
interface RAGResponse {
  answer: string;
  confidence: 'high' | 'medium' | 'low' | 'none';
  sources: Array<{
    title: string;
    url: string;
    chunk: string;
    relevance: number;
  }>;
  isModelSupplement: boolean;
  category: string; // 问题所属分类
}
```

#### 1.3 上下文工程

**参考 jike-100city-main 的架构**：

```

├── user-info.ts          # <user_info> 构建器

```

**XML 结构化提示词**：
```xml
<system>
  <role>你是 Frontend Career Agent，专注前端领域的 AI 助手</role>
  <capabilities>
    <tool name="rag_search">前端知识库检索</tool>
    <tool name="web_search">联网搜索</tool>
  </capabilities>
  <rules>
    <rule>优先使用知识库回答</rule>
    <rule>回答必须标注来源</rule>
    <rule>没有依据时明确说明</rule>
  </rules>
</system>

<user_info>
  <current_time>2026-05-09 14:30</current_time>
  <response_style>技术严谨</response_style>
</user_info>

<user_query>
React Fiber 是什么？
</user_query>
```

**首轮/后续轮区分**：
- 首轮：完整注入用户偏好、系统规则
- 后续轮：简化提醒 `<user_preference>请参考之前设置的回答风格</user_preference>`

#### 2.2 简历评分

**评分维度**：
```typescript
interface ResumeScore {
  total: number; // 0-100
  dimensions: {
    frontendFoundation: number;  // 前端基础
    engineering: number;         // 工程化能力
    projectDepth: number;        // 项目深度
    businessValue: number;       // 业务价值
    aiExperience: number;        // AI 应用经验
    readability: number;         // 可读性
  };
}
```

**评分标准**（需要细化）：
- 前端基础：技术栈覆盖度、深度
- 工程化：构建、部署、CI/CD、代码规范
- 项目深度：复杂度、技术挑战、解决方案
- 业务价值：量化指标、业务结果
- AI 经验：AI 应用、Agent、RAG 等
- 可读性：结构清晰、重点突出

## 风险和挑战

1. **RAG 准确性** - 检索质量直接影响回答质量，需要持续优化
2. **知识库覆盖** - 前端知识面广，需要持续补充内容
3. **评分标准** - 简历评分标准需要反复打磨，避免主观
4. **用户预期** - 需要明确产品边界，避免用户期望过高

## 附录

### 竞品参考
- ChatGPT：通用对话，无垂直深度
- Perplexity：搜索 + 引用，但不垂直
- 各类简历优化工具：功能单一，无 AI 深度分析

### 差异化
- 垂直前端领域，深度 > 广度
- RAG 有引用、有置信度、不乱编
- 从学习到求职全链路覆盖
- 开发者自己就是用户，能持续迭代
