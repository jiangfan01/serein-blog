# Virtual DOM

**Virtual DOM（虚拟 DOM）** 是 React 引入的一种提高 UI 渲染性能的机制。

## Virtual DOM 是什么？

Virtual DOM 是一种以 JavaScript 对象的形式描述 DOM 结构的技术，它是对真实 DOM 的一种抽象表示。

> 通俗理解：Virtual DOM 就是一棵以 JS 对象表示的"虚拟 DOM 树"。

```js
// 真实 DOM
<div class="container">
  <h1>Hello</h1>
</div>

// Virtual DOM（JS 对象）
{
  type: 'div',
  props: { className: 'container' },
  children: [
    {
      type: 'h1',
      props: {},
      children: ['Hello']
    }
  ]
}
```

## 工作原理（核心流程）

### 1. 初始渲染

- React 通过 `render()` 方法构建一棵 Virtual DOM 树
- React 将其映射为真实 DOM，插入到页面中

### 2. 状态更新时

- React 会重新构建一棵新的 Virtual DOM 树
- React 将新旧 Virtual DOM 树 **进行对比（diff）**
- 找出变化的部分（称为 **差异 diff**）
- React 只对 **需要更新的节点** 进行 DOM 操作

### 3. 更新 DOM

根据 diff 结果，React 通过内部的 **Reconciliation（协调算法）**，高效地将变更同步到真实 DOM。

## Diff 算法策略

React 的 diff 算法基于以下假设进行优化：

1. **同层比较** - 只比较同一层级的节点
2. **类型判断** - 不同类型的元素会产生不同的树
3. **Key 标识** - 通过 key 来标识列表中的元素

```jsx
// 使用 key 帮助 React 识别列表项
{items.map(item => (
  <li key={item.id}>{item.name}</li>
))}
```

## 为什么不直接操作真实 DOM？

| 问题 | 说明 |
|------|------|
| 性能开销大 | 真实 DOM 操作昂贵，涉及重排重绘 |
| 频繁更新 | 多次 setState 会引发多次渲染 |
| 跨平台限制 | 直接操作 DOM 无法支持其他平台 |

## Virtual DOM 的优势

| 优势 | 说明 |
|------|------|
| 批量更新 | 合并多次更新，减少 DOM 操作 |
| 最小化更新 | 只更新变化的部分 |
| 跨平台 | 支持 React Native、SSR 等 |
| 声明式编程 | 开发者只需关注状态，不用手动操作 DOM |

## 示例：状态更新流程

```jsx
function Counter() {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        +1
      </button>
    </div>
  );
}
```

当点击按钮时：

1. `setCount` 触发状态更新
2. React 创建新的 Virtual DOM 树
3. Diff 算法比较新旧树，发现 `<p>` 内容变化
4. 只更新 `<p>` 元素的文本内容
5. 真实 DOM 完成最小化更新

## 总结

- Virtual DOM 是真实 DOM 的 JS 对象表示
- 通过 Diff 算法找出最小变更
- 批量、最小化地更新真实 DOM
- 提供了声明式编程和跨平台能力
