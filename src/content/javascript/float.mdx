# JavaScript 的浮点精度问题

## 什么是浮点精度问题？

JavaScript 中的所有数字（包括整数和小数）都基于 **IEEE 754 双精度浮点数** 规范实现，这会导致一些十进制数字无法被精确表示。

最常见的问题是：

```js
0.1 + 0.2 === 0.3 // false
0.1 + 0.2         // 0.30000000000000004
```

## 为什么会出现精度误差？

- 计算机使用**二进制**表示所有数据，但像 0.1、0.2 这样的十进制小数在二进制中是**无限循环小数**
- 浮点数存储是有精度限制的（最多 64 位），最终被**截断**，造成误差

## 常见解决方案

### 1. 使用 toFixed() 四舍五入

```js
let result = Number((0.1 + 0.2).toFixed(2)); // 0.3
```

> ⚠️ 注意：`toFixed()` 返回字符串，需用 `Number()` 转换回数字。

### 2. 使用 Number.EPSILON 误差容差判断

```js
function isEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}

isEqual(0.1 + 0.2, 0.3); // true
```

### 3. 整数放大法（适用于金额等精度敏感场景）

```js
function add(a, b) {
  const factor = 10 ** 10;
  return (Math.round(a * factor) + Math.round(b * factor)) / factor;
}
```

### 4. 使用高精度库（如 decimal.js）

```bash
npm install decimal.js
```

```js
import Decimal from "decimal.js";

const result = new Decimal(0.1).plus(0.2);
console.log(result.toString()); // "0.3"
```

## 常见出错场景

| 场景描述 | 示例 | 是否出错 |
|----------|------|----------|
| 小数相加 | `0.1 + 0.2` | ❌ |
| 金额计算 | `199.99 * 100` | ❌ |
| 数组下标判断 | `arr[0.1 + 0.2]` | ❌ |
| BigInt 用于浮点 | `BigInt(0.1)` | ❌ 不支持 |

## 最佳实践总结

- 金额、权重、百分比等精度敏感数据：**使用整数处理或高精度库**
- 比较两个浮点数是否相等：**使用误差容差法**
- 展示数据时保留几位小数：**使用 `toFixed(n)`、`Intl.NumberFormat`**

## 常见对比方案

| 方法 | 原理 | 是否推荐 | 使用场景 |
|------|------|----------|----------|
| `toFixed()` | 保留小数，四舍五入 | ✅ | 展示为主 |
| `Number.EPSILON` | 容差判断是否接近 | ✅ | 判断是否相等 |
| 整数放大法 | 放大避免浮点 | ✅ | 金额等运算 |
| 精度库（decimal） | 精确封装内部浮点机制 | ✅✅ | 金融、账务、高精度运算 |
