# 路径计数（动态规划）

## 题目描述

给定一个 `m × n` 的棋盘，骑士只能向下或向右移动一步，求从左上角到右下角的不同路径数。

### 示例

```js
uniquePaths(3, 3); // 输出 6
```

## 解法一：从起点开始的动态规划（迭代）

```js
function uniquePaths(m, n) {
  // 创建DP表 (m x n的二维数组)
  const dp = [];
  
  // 遍历每一行
  for (let i = 0; i < m; i++) {
    // 初始化当前行为空数组
    dp.push([]);
    
    // 遍历当前行的每一列
    for (let j = 0; j < n; j++) {
      // 边界情况：第一行或第一列
      if (i === 0 || j === 0) {
        // 对于第一行或第一列，只有一条直线路径
        dp[i][j] = 1; 
      } else {
        // 状态转移方程：
        // 到达(i,j)的路径数 = 从左边的路径数 + 从上边的路径数
        dp[i][j] = dp[i][j - 1] + dp[i - 1][j];
      }
    }
  }
  
  // 返回右下角位置的路径总数
  return dp[m - 1][n - 1];
}
```

## 解法二：从终点逆推的递归 + 记忆化

```js
function uniquePathsReverse(m, n) {
  const memo = Array.from({ length: m }, () => Array(n).fill(-1));

  function dfs(i, j) {
    if (i === m - 1 && j === n - 1) return 1; // 到达终点
    if (memo[i][j] !== -1) return memo[i][j];

    let paths = 0;
    if (i + 1 < m) paths += dfs(i + 1, j); // 向下走
    if (j + 1 < n) paths += dfs(i, j + 1); // 向右走

    memo[i][j] = paths;
    return paths;
  }

  return dfs(0, 0);
}
```

## 思路解析

- **解法一**：用二维数组自底向上迭代计算，每个格子的路径数等于上方和左方路径数之和
- **解法二**：递归从起点出发尝试向下和向右走，用记忆化缓存结果避免重复计算
- 两种方法本质相同，递归版本更直观易懂，迭代版本更高效
- 时间复杂度均为 `O(m × n)`
- 空间复杂度：递归栈 `O(m+n)` + 记忆数组 `O(m × n)`
