# 带并发限制的 Promise 请求池

> 这个问题不是为了手写算法而手写。真实场景是：用户一次上传很多文档，前端不能把所有切片一口气全打到后端。

## 问题场景

### 真实需求

在 AI 知识库项目中，用户上传了 100 份 PDF 文档，前端需要：

1. 将每份文档切片（chunk）
2. 调用后端 Embedding 接口生成向量
3. 存储到向量数据库

### 最容易写出来的版本

```typescript
// ❌ 直接使用 Promise.all()
const tasks = files.map(file => uploadFile(file));
await Promise.all(tasks);  // 瞬间发出 100 个请求！
```

**问题**：

1. **浏览器并发限制** - Chrome 对同一域名最多 6 个并发连接
2. **后端压力** - 瞬间 100 个请求可能触发网关限流
3. **内存占用** - 100 个文件同时读取到内存
4. **用户体验** - 无法显示进度，失败后难以恢复

### 更稳一点的做法

**需要一个并发控制器**，始终保持最多 `max` 个任务在运行：

```typescript
// ✅ 使用并发控制
await asyncPool(5, files, async (file) => {
  return uploadFile(file);
});
```

---

## 思路

### 1. 使用 Promise.race 动态竞争

关键点是维护一个正在执行的池子：

```typescript
// 维护一个执行池（最多 max 个 Promise）
const executing = [];

// 当池子满了，用 Promise.race 等待任何一个完成
await Promise.race(executing);

// 有一个完成了，池子就有空位了，可以加入新任务
```

### 2. 两个指针管理任务

```typescript
let currentIndex = 0;  // 当前要执行的任务索引
const executing = [];  // 正在执行的 Promise 池（最多 max 个）

// 循环逻辑：
// 1. 如果 currentIndex < tasks.length，继续添加任务
// 2. 如果 executing.length >= max，等待一个完成
// 3. 重复直到所有任务完成
```

### 3. 流程图

```
任务队列: [Task0, Task1, Task2, Task3, Task4, Task5, ...]
并发数: max = 3

初始状态:
  executing: []
  currentIndex: 0

第 1 步: 填充执行池
  executing: [Task0, Task1, Task2]
  currentIndex: 3

第 2 步: Task1 完成
  executing: [Task0, Task2]  (移除 Task1)
  立即补充: executing: [Task0, Task2, Task3]
  currentIndex: 4

第 3 步: Task0 完成
  executing: [Task2, Task3]
  立即补充: executing: [Task2, Task3, Task4]
  currentIndex: 5

...重复直到所有任务完成
```

## 实现方案

### 方案 1：基础版（Promise.race）

```typescript
/**
 * 并发控制的异步任务池
 * @param limit 最大并发数
 * @param tasks 任务数组
 * @param iteratorFn 任务执行函数
 */
async function asyncPool<T, R>(
  limit: number,
  tasks: T[],
  iteratorFn: (task: T, index: number) => Promise<R>
): Promise<R[]> {
  const results: R[] = [];           // 存储所有结果
  const executing: Promise<void>[] = []; // 正在执行的 Promise 池
  
  for (let i = 0; i < tasks.length; i++) {
    const task = tasks[i];
    
    // 创建一个 Promise 包装器
    const p = Promise.resolve().then(() => iteratorFn(task, i));
    
    // 保存结果
    results.push(p as any);
    
    // 如果任务数小于并发数，不需要等待
    if (tasks.length >= limit) {
      // 创建一个执行完成后从池中移除的 Promise
      const e = p.then(() => {
        // 任务完成，从执行池中移除
        executing.splice(executing.indexOf(e), 1);
      });
      
      executing.push(e);
      
      // 如果执行池满了，等待任何一个完成
      if (executing.length >= limit) {
        await Promise.race(executing);
      }
    }
  }
  
  // 等待所有任务完成
  return Promise.all(results);
}
```

**使用示例**：

```typescript
// 模拟上传文件
async function uploadFile(file: File): Promise<string> {
  await new Promise(resolve => setTimeout(resolve, 1000));
  return `Uploaded: ${file.name}`;
}

// 100 个文件，最多 5 个并发

  return uploadFile(file);
});

console.log(results);
```

### 方案 2：增强版（支持进度、错误处理、重试）

```typescript
interface AsyncPoolOptions<T, R> {
  limit: number;                                    // 最大并发数
  tasks: T[];                                       // 任务数组
  iteratorFn: (task: T, index: number) => Promise<R>;  // 任务执行函数
  onProgress?: (completed: number, total: number) => void;  // 进度回调
  onError?: (error: Error, task: T, index: number) => void; // 错误回调
  retryTimes?: number;                              // 重试次数
  retryDelay?: number;                              // 重试延迟（ms）
}

interface AsyncPoolResult<R> {
  results: (R | null)[];      // 结果数组（失败的为 null）
  errors: Array<{ index: number; error: Error }>;  // 错误列表
  successCount: number;       // 成功数量
  failCount: number;          // 失败数量
}

async function asyncPoolEnhanced<T, R>(
  options: AsyncPoolOptions<T, R>
): Promise<AsyncPoolResult<R>> {
  const {
    limit,
    tasks,
    iteratorFn,
    onProgress,
    onError,
    retryTimes = 0,
    retryDelay = 1000,
  } = options;
  
  const results: (R | null)[] = new Array(tasks.length).fill(null);
  const errors: Array<{ index: number; error: Error }> = [];
  const executing: Promise<void>[] = [];
  
  let completed = 0;
  let successCount = 0;
  let failCount = 0;
  
  // 重试逻辑
  async function executeWithRetry(
    task: T,
    index: number,
    retriesLeft: number
  ): Promise<R> {
    try {
      return await iteratorFn(task, index);
    } catch (error) {
      if (retriesLeft > 0) {
        // 等待后重试
        await new Promise(resolve => setTimeout(resolve, retryDelay));
        return executeWithRetry(task, index, retriesLeft - 1);
      }
      throw error;
    }
  }
  
  for (let i = 0; i < tasks.length; i++) {
    const task = tasks[i];
    
    const p = executeWithRetry(task, i, retryTimes)
      .then(result => {
        results[i] = result;
        successCount++;
      })
      .catch(error => {
        results[i] = null;
        errors.push({ index: i, error });
        failCount++;
        onError?.(error, task, i);
      })
      .finally(() => {
        completed++;
        onProgress?.(completed, tasks.length);
      });
    
    if (tasks.length >= limit) {
      const e = p.then(() => {
        executing.splice(executing.indexOf(e), 1);
      });
      
      executing.push(e);
      
      if (executing.length >= limit) {
        await Promise.race(executing);
      }
    }
  }
  
  // 等待所有任务完成
  await Promise.all(executing);
  
  return {
    results,
    errors,
    successCount,
    failCount,
  };
}
```

**使用示例**：

```typescript
const result = await asyncPoolEnhanced({
  limit: 5,
  tasks: files,
  iteratorFn: async (file) => {
    return uploadFile(file);
  },
  onProgress: (completed, total) => {
    console.log(`进度: ${completed}/${total} (${Math.round(completed / total * 100)}%)`);
  },
  onError: (error, file, index) => {
    console.error(`文件 ${file.name} 上传失败:`, error);
  },
  retryTimes: 3,
  retryDelay: 1000,
});

console.log(`成功: ${result.successCount}, 失败: ${result.failCount}`);
```

### 方案 3：支持暂停/恢复的版本

```typescript
class AsyncPoolController<T, R> {
  private limit: number;
  private tasks: T[];
  private iteratorFn: (task: T, index: number) => Promise<R>;
  private results: (R | null)[] = [];
  private executing: Set<Promise<void>> = new Set();
  private currentIndex = 0;
  private isPaused = false;
  private pausePromise: Promise<void> | null = null;
  private pauseResolve: (() => void) | null = null;
  
  constructor(
    limit: number,
    tasks: T[],
    iteratorFn: (task: T, index: number) => Promise<R>
  ) {
    this.limit = limit;
    this.tasks = tasks;
    this.iteratorFn = iteratorFn;
    this.results = new Array(tasks.length).fill(null);
  }
  
  async run(): Promise<(R | null)[]> {
    while (this.currentIndex < this.tasks.length || this.executing.size > 0) {
      // 检查是否暂停
      if (this.isPaused && this.pausePromise) {
        await this.pausePromise;
      }
      
      // 填充执行池
      while (
        this.currentIndex < this.tasks.length &&
        this.executing.size < this.limit
      ) {
        const index = this.currentIndex++;
        const task = this.tasks[index];
        
        const p = this.executeTask(task, index);
        this.executing.add(p);
        
        p.finally(() => {
          this.executing.delete(p);
        });
      }
      
      // 等待任意一个完成
      if (this.executing.size > 0) {
        await Promise.race(this.executing);
      }
    }
    
    return this.results;
  }
  
  private async executeTask(task: T, index: number): Promise<void> {
    try {
      const result = await this.iteratorFn(task, index);
      this.results[index] = result;
    } catch (error) {
      this.results[index] = null;
      console.error(`Task ${index} failed:`, error);
    }
  }
  
  pause(): void {
    if (!this.isPaused) {
      this.isPaused = true;
      this.pausePromise = new Promise(resolve => {
        this.pauseResolve = resolve;
      });
    }
  }
  
  resume(): void {
    if (this.isPaused) {
      this.isPaused = false;
      this.pauseResolve?.();
      this.pausePromise = null;
      this.pauseResolve = null;
    }
  }
  
  getProgress(): { completed: number; total: number; percentage: number } {
    const completed = this.results.filter(r => r !== null).length;
    const total = this.tasks.length;
    return {
      completed,
      total,
      percentage: Math.round((completed / total) * 100),
    };
  }
}
```

**使用示例**：

```typescript
const controller = new AsyncPoolController(5, files, async (file) => {
  return uploadFile(file);
});

// 开始执行
const resultsPromise = controller.run();

// 5 秒后暂停
setTimeout(() => {
  controller.pause();
  console.log('暂停上传');
}, 5000);

// 10 秒后恢复
setTimeout(() => {
  controller.resume();
  console.log('恢复上传');
}, 10000);

// 定时查看进度
const timer = setInterval(() => {
  const progress = controller.getProgress();
  console.log(`进度: ${progress.percentage}%`);
}, 1000);

const results = await resultsPromise;
clearInterval(timer);
```

## 放到 PDF 文档切片上传里

### 上传流程

```typescript
interface UploadChunk {
  file: File;
  chunkIndex: number;
  totalChunks: number;
  chunk: Blob;
}

interface UploadResult {
  fileId: string;
  chunkIndex: number;
  success: boolean;
}

class DocumentUploader {
  private chunkSize = 1024 * 1024; // 1MB per chunk
  private maxConcurrency = 5;
  
  /**
   * 将文件切片
   */
  private createChunks(file: File): UploadChunk[] {
    const chunks: UploadChunk[] = [];
    const totalChunks = Math.ceil(file.size / this.chunkSize);
    
    for (let i = 0; i < totalChunks; i++) {
      const start = i * this.chunkSize;
      const end = Math.min(start + this.chunkSize, file.size);
      const chunk = file.slice(start, end);
      
      chunks.push({
        file,
        chunkIndex: i,
        totalChunks,
        chunk,
      });
    }
    
    return chunks;
  }
  
  /**
   * 上传单个切片
   */
  private async uploadChunk(chunk: UploadChunk): Promise<UploadResult> {
    const formData = new FormData();
    formData.append('file', chunk.chunk);
    formData.append('fileName', chunk.file.name);
    formData.append('chunkIndex', chunk.chunkIndex.toString());
    formData.append('totalChunks', chunk.totalChunks.toString());
    
    const response = await fetch('/api/upload/chunk', {
      method: 'POST',
      body: formData,
    });
    
    if (!response.ok) {
      throw new Error(`Upload failed: ${response.statusText}`);
    }
    
    const data = await response.json();
    return {
      fileId: data.fileId,
      chunkIndex: chunk.chunkIndex,
      success: true,
    };
  }
  
  /**
   * 上传多个文件（带并发控制）
   */
  async uploadFiles(
    files: File[],
    onProgress?: (progress: { completed: number; total: number }) => void
  ): Promise<Map<string, UploadResult[]>> {
    // 1. 将所有文件切片
    const allChunks: UploadChunk[] = [];
    for (const file of files) {
      const chunks = this.createChunks(file);
      allChunks.push(...chunks);
    }
    
    console.log(`总共 ${allChunks.length} 个切片需要上传`);
    
    // 2. 使用并发控制上传
    const result = await asyncPoolEnhanced({
      limit: this.maxConcurrency,
      tasks: allChunks,
      iteratorFn: async (chunk) => {
        return this.uploadChunk(chunk);
      },
      onProgress: (completed, total) => {
        onProgress?.({ completed, total });
      },
      onError: (error, chunk) => {
        console.error(
          `文件 ${chunk.file.name} 的第 ${chunk.chunkIndex} 个切片上传失败:`,
          error
        );
      },
      retryTimes: 3,
      retryDelay: 1000,
    });
    
    // 3. 按文件分组结果
    const resultsByFile = new Map<string, UploadResult[]>();
    result.results.forEach((uploadResult, index) => {
      if (uploadResult) {
        const chunk = allChunks[index];
        const fileName = chunk.file.name;
        
        if (!resultsByFile.has(fileName)) {
          resultsByFile.set(fileName, []);
        }
        
        resultsByFile.get(fileName)!.push(uploadResult);
      }
    });
    
    return resultsByFile;
  }
  
  /**
   * 合并切片（通知后端）
   */
  async mergeChunks(fileId: string, totalChunks: number): Promise<void> {
    const response = await fetch('/api/upload/merge', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ fileId, totalChunks }),
    });
    
    if (!response.ok) {
      throw new Error(`Merge failed: ${response.statusText}`);
    }
  }
}
```

**使用示例**：

```typescript
// React 组件中使用
function DocumentUploadPage() {
  const [progress, setProgress] = useState({ completed: 0, total: 0 });
  const [uploading, setUploading] = useState(false);
  
  const handleUpload = async (files: File[]) => {
    setUploading(true);
    
    const uploader = new DocumentUploader();
    
    try {
      // 上传所有切片
      const results = await uploader.uploadFiles(files, (progress) => {
        setProgress(progress);
      });
      
      // 合并切片
      for (const [fileName, chunks] of results) {
        const fileId = chunks[0].fileId;
        await uploader.mergeChunks(fileId, chunks.length);
        console.log(`文件 ${fileName} 上传完成`);
      }
      
      alert('所有文件上传成功！');
    } catch (error) {
      console.error('上传失败:', error);
      alert('上传失败，请重试');
    } finally {
      setUploading(false);
    }
  };
  
  return (
    <div>
      <input
        type="file"
        multiple
        accept=".pdf"
        onChange={(e) => {
          const files = Array.from(e.target.files || []);
          handleUpload(files);
        }}
      />
      
      {uploading && (
        <div>
          <p>
            上传进度: {progress.completed} / {progress.total} (
            {Math.round((progress.completed / progress.total) * 100)}%)
          </p>
          <progress value={progress.completed} max={progress.total} />
        </div>
      )}
    </div>
  );
}
```

## 回头看几个关键点

### 1. Promise.race 的妙用

```typescript
// Promise.race 返回最先完成的 Promise
const executing = [promise1, promise2, promise3];
await Promise.race(executing);  // 等待任意一个完成
```

### 2. 动态维护执行池

```typescript
// 任务完成后，从执行池中移除
const e = p.then(() => {
  executing.splice(executing.indexOf(e), 1);
});
```

### 3. 两个指针的作用

- `currentIndex` - 记录下一个要执行的任务索引
- `executing` - 记录正在执行的 Promise（最多 max 个）

### 4. HTTP/2 的优势

你提到的 HTTP/2 确实可以创建更多的并发连接：

```typescript
// HTTP/1.1: 浏览器限制 6 个并发
// HTTP/2: 可以复用同一个 TCP 连接，支持多路复用

// 但即使用 HTTP/2，也建议控制并发数：
// 1. 避免后端压力过大
// 2. 避免触发网关限流
// 3. 更好的错误处理和进度显示
```

### 5. 失败恢复策略

```typescript
// 记录失败的切片
const failedChunks: UploadChunk[] = [];

// 重试失败的切片
if (failedChunks.length > 0) {
  await asyncPool(5, failedChunks, async (chunk) => {
    return uploadChunk(chunk);
  });
}
```

## 性能对比

### 测试场景

- 100 个文件
- 每个文件 10MB
- 切片大小 1MB
- 总共 1000 个切片

### 不同方案的表现

| 方案 | 并发数 | 总耗时 | 内存占用 | 成功率 |
|------|--------|--------|----------|--------|
| Promise.all() | 1000 | 10s | 1GB | 60% (网关限流) |
| asyncPool (limit=5) | 5 | 200s | 50MB | 100% |
| asyncPool (limit=10) | 10 | 100s | 100MB | 100% |
| asyncPool (limit=20) | 20 | 50s | 200MB | 95% (偶尔限流) |

这个表只是一个参考，实际并发数还是要看文件大小、接口耗时、网关限流和用户网络。一般我会从 5 或 10 开始压测。

## 如果要讲给别人听

### 思路

> 使用 `Promise.race` 动态维护一个执行池，始终保持最多 `max` 个任务在运行。当有任务完成时，立即从池中移除，并补充新任务，直到所有任务完成。

### 关键代码

```typescript
const executing = [];

for (let i = 0; i < tasks.length; i++) {
  const p = executeTask(tasks[i]);
  
  const e = p.then(() => {
    executing.splice(executing.indexOf(e), 1);
  });
  
  executing.push(e);
  
  if (executing.length >= limit) {
    await Promise.race(executing);  // 等待任意一个完成
  }
}

await Promise.all(executing);  // 等待剩余任务
```

### 扩展问题

**Q: 为什么不用 Promise.allSettled()？**

A: `Promise.allSettled()` 会等待所有 Promise 完成，无法实现动态补充任务。

**Q: 如何处理失败的任务？**

A: 可以在 `catch` 中记录失败的任务，最后统一重试。

**Q: 如何实现优先级队列？**

A: 可以先对任务排序，或者使用堆（Heap）数据结构。
