会话摘要压缩（含"摘要膨胀"防治）

每发一轮,历史 token 涨。到 80% 预算触发压缩:把最近 3 轮之外的旧对话压成摘要。关键开关:摘要上限。关掉它,看摘要怎么膨胀到炸;打开它,看它定长稳如老狗。

开
点"发一轮对话",看历史怎么涨、到 80% 怎么压。试试把"摘要上限"关掉再发几轮。
对照真实代码 summarizer.ts：感知 = 估算 摘要token + 截断点后消息；触发 = ≥ 80% × conversationHistory 预算；压缩 = 最近 3 轮之外的旧对话 + 旧摘要 → 融合成新摘要，写回 ChatSession.summaryContent + summarizedUpToMessageId(截断点)。
上限那一栏就是你逮到的坑：没有上限，摘要每次融合越滚越大，最终比原文还大 → 炸。MAX_SUMMARY_TOKENS 让摘要定长，永不膨胀。
