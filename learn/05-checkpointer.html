每次 invoke(input, { configurable: { thread_id } }) 的内部动作：① 按 thread_id 从存档取出上次 state → ② append 新消息 → ③ 跑图 → ④ 把新 state 存回该 thread_id。
所以同一个 thread 记忆累积，换 thread_id 就是另一份存档。MemorySaver 存内存（重启丢）；换 PostgresSaver 就跨进程跨重启持久化 = 真正的断点续传。对照你的 ChatExecution 表 / Eino 的 CheckPointStore。
