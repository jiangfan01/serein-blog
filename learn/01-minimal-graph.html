LangGraph 从零 · L1

一张图是怎么跑起来的

没有 LLM，没有魔法。只有三样东西：流过节点的 State、改 State 的 节点、决定"多次写入怎么合并"的 reducer。点"单步执行"，看 state 怎么一步步变。

LangGraph 的零件解剖

这是一张真实的 Agent 图（带条件边和环，就是你 agent.ts 的结构）。对着编号看每个零件是什么。

实线 = 普通边（固定走向） 虚线 = 条件边（读 State 决定） 绿色回边 = 环
1

边界节点 · START

图的入口。LangGraph 内置，不用你写逻辑，只标记"从这开始"。

2

节点 · Node（agent）

真正干活的函数。这里内部调用 LLM。输入完整 State，返回对 State 的局部更新。

3

条件边 · Conditional Edge

读 State 决定走哪条路：上一条消息有 tool_calls 就去 tools，没有就去 END。对应你的 shouldContinue。

4

节点 · Node（tools）

另一个节点，执行工具调用。执行完把结果写回 State。

5

环 · Cycle（回边）

tools 执行完又回到 agent。这条回边让 Agent 能"调完工具再想、再调"。它是 Agent 的灵魂，也是 LangGraph 不是 DAG 的原因。

6

边界节点 · END

出口。走到这里整张图结束，返回最终 State。

7

State · 共享数据

贯穿全程。每个节点都读它、改它（按 reducer 合并）。它不含逻辑，只是流动的数据。
